# Intent（開発意図）

## プロジェクト名
jailrun v0.1.0 — コードベース整理・品質向上・ドキュメント整備

## 開発の目的
既存コードベースを全体的にチェック・整理し、ドキュメントを利用に耐える状態にアップデートする。jailrunはAIコーディングエージェント向けのセキュリティラッパー（認証情報分離 + OSサンドボックス）であり、初回リリースに向けてコード品質・テストカバレッジ・ドキュメントの3つの軸で品質を引き上げる。

## ターゲットユーザー
- **エンドユーザー**: AIコーディングエージェント（Codex, Claude等）を安全に実行したい開発者
- **コントリビューター**: jailrunの開発に参加する開発者

## ビジネス価値
- 初回利用者がREADMEを読んですぐに導入・利用できる状態を実現
- コード品質向上によりバグリスクを低減し、今後の機能追加を容易にする
- テストカバレッジ向上により変更時のリグレッションリスクを低減する

## 成功基準
- 全シェルスクリプトがPOSIX準拠テストをパスする（既存テスト維持）
- 主要コンポーネントのテストカバレッジギャップが解消される（credential-guard, proxy.py, Linux systemd等）
- README.mdから導入・基本利用・設定までの手順が明確に辿れる
- 開発者向けドキュメント（アーキテクチャ、コントリビューションガイド）が整備される
- `config.py`の責務分割が完了する
- .gitignoreが整備され、不要ファイル（__pycache__等）が除外される

## 含まれるもの
1. **コード品質改善（リファクタリング）**
   - `config.py`の責務分割（TOML解析、CLI、マイグレーション）
   - `sandbox.sh`の関心分離
   - `token.sh`のCLI解析/ロジック分離
   - 命名統一・不要コード削除
2. **テスト追加・整備**
   - credential-guard, credentials, proxy.py, Linux systemd, keychain等のテスト追加
   - エラー条件・エッジケースのテスト強化
3. **ディレクトリ構成の見直し**
   - .gitignore整備（__pycache__除外等）
   - ファイル配置の最適化
4. **ドキュメント整備**
   - README.md と docs/README.md の整理・統合
   - ユーザー向けドキュメント（導入、利用方法、設定）
   - 開発者向けドキュメント（アーキテクチャ、コントリビューションガイド）

## 含まれないもの
- 新機能の追加
- 対応エージェントの拡張
- パフォーマンス最適化
- CI/CDパイプラインの構築

## 期限とマイルストーン
特に期限なし。品質最優先。

## 制約事項
- POSIX sh準拠を維持すること（bash拡張禁止）
- 既存のテストがすべてパスし続けること
- macOS / Linux 両プラットフォームのサポートを維持すること
- Python 3依存は config.py / proxy.py に限定すること
