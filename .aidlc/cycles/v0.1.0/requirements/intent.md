# Intent（開発意図）

## プロジェクト名
jailrun v0.1.0 — コードベース整理・品質向上・ドキュメント整備

## 開発の目的
既存コードベースを全体的にチェック・整理し、ドキュメントを利用に耐える状態にアップデートする。jailrunはAIコーディングエージェント向けのセキュリティラッパー（認証情報分離 + OSサンドボックス）であり、初回リリースに向けてコード品質・テストカバレッジ・ドキュメントの3つの軸で品質を引き上げる。

## ターゲットユーザー
- **エンドユーザー**: AIコーディングエージェント（Codex, Claude等）を安全に実行したい開発者
- **コントリビューター**: jailrunの開発に参加する開発者

## ビジネス価値
- 初回利用者がREADMEを読んですぐに導入・利用できる状態を実現
- コード品質向上によりバグリスクを低減し、今後の機能追加を容易にする
- テストカバレッジ向上により変更時のリグレッションリスクを低減する

## 成功基準
- 全シェルスクリプトが既存のPOSIX準拠テスト（`tests/posix_compliance.bats`）をパスする
- 以下のコンポーネントに対するテストファイルが追加される: `proxy.py`, `sandbox-linux-systemd.sh`
- `config.py` が以下の3ファイルに分割される: `lib/config.py`（TOML解析・設定値の読み書き）、`lib/config_cli.py`（show/set/edit/init等のCLIコマンド処理）、`lib/config_migrate.py`（レガシー設定からTOMLへのマイグレーション）
- README.md に以下の章が含まれる: インストール手順、クイックスタート、設定リファレンス、トラブルシューティング
- docs/ に開発者向けドキュメント（アーキテクチャ概要、コントリビューションガイド）が存在する
- .gitignore に `__pycache__/` が含まれ、リポジトリに `.pyc` ファイルが存在しない
- `make test` が全テストパスで終了する

## 含まれるもの

### Must（必達）
1. **コード品質改善（リファクタリング）**
   - `config.py` の責務分割: `lib/config.py`（TOML解析）、`lib/config_cli.py`（CLIコマンド処理）、`lib/config_migrate.py`（マイグレーション）に分割
   - .gitignore整備（`__pycache__/`, `*.pyc` 除外）
   - `lib/__pycache__/` がリポジトリに含まれている場合は削除
2. **テスト追加**
   - `proxy.py` のユニットテスト（ドメインフィルタリング、DNSリバインディング検出）
   - `sandbox-linux-systemd.sh` のプロパティ生成テスト
3. **ドキュメント整備**
   - README.md の再構成（インストール、クイックスタート、設定リファレンス、トラブルシューティング）
   - docs/README.md との内容重複を解消

### Should（推奨）
4. **追加リファクタリング**
   - `sandbox.sh` の関心分離（プラットフォーム共通ロジック / env-spec生成 / プロキシ管理）
   - `token.sh` のCLI解析とキーチェーン操作の分離
5. **追加テスト**
   - `credential-guard.sh` の統合テスト
   - エラー条件テスト（設定ファイル不正、バイナリ不在等）
6. **追加ドキュメント**
   - docs/architecture.md（アーキテクチャ概要、データフロー図）
   - docs/contributing.md（開発環境セットアップ、テスト実行方法、コーディング規約）

### Can（余裕があれば）
7. **命名統一・不要コード削除**
   - 全ファイルの命名規約チェックと統一
8. **GitHub PAT設定ガイドの更新**
   - docs/github-pat-setup.md の最新化

## 含まれないもの
- **新機能の追加**: 新しいエージェント対応、新しいサンドボックスバックエンド、新しい設定項目の追加は含まない。ただし、既存機能の使い勝手を向上させる変更（エラーメッセージ改善、ヘルプテキスト充実等）はリファクタリングの範囲として許容する
- 対応エージェントの拡張
- パフォーマンス最適化
- CI/CDパイプラインの構築

## 後方互換性要件
- **CLI引数互換**: `jailrun <agent> [args]` の引数体系を変更しない
- **設定ファイル互換**: `~/.config/jailrun/config.toml` のキー名・構造を変更しない。`config.py` 分割後も同一の設定ファイルを読み書きできること
- **トークン互換**: キーチェーンに保存済みのトークンが引き続き利用可能であること
- **シム互換**: `lib/shims/codex` の動作を変更しない
- **ディレクトリ構成**: `bin/`, `lib/`, `lib/platform/`, `lib/shims/`, `tests/`, `docs/` のトップレベル構成を維持する

## 期限とマイルストーン
期限なし。品質最優先。

- **マイルストーン1**: Must項目完了（config.py分割、必須テスト追加、README再構成）
- **マイルストーン2**: Should項目完了（追加リファクタリング、追加テスト、開発者ドキュメント）
- **マイルストーン3**: Can項目完了（命名統一、ドキュメント最新化）

## 制約事項
- POSIX sh準拠を維持すること（bash拡張禁止）
- 既存のテストがすべてパスし続けること
- macOS / Linux 両プラットフォームのサポートを維持すること
- Python 3依存は lib/ 配下の .py ファイルに限定すること
